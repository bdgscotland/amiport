# cd.device/CD_ADDFRAMEINT

> *Source: Amiga Developer CD v2.1. (C) Commodore-Amiga / Amiga Inc.*

   NAME

       CD_ADDFRAMEINT -- add a CD-frame software interrupt handler.
   IO REQUEST

```c
       io_Device       preset by the call to [OpenDevice()](autodocs-3.5/cd-device-opendevice.md)
       io_Unit         preset by the call to [OpenDevice()](autodocs-3.5/cd-device-opendevice.md)
       io_Command      CD_ADDFRAMEINT
       io_Length       sizeof(struct Interrupt)
       io_Data         pointer to [Interrupt](autodocs-3.5/include-exec-interrupts-h.md) structure
```
   RESULTS

```c
       io_Error        0 for success, or an error code as defined in
                       [<devices/cd.h>](autodocs-3.5/include-devices-cd-h.md)
```
   FUNCTION

```c
       This command lets you add a software interrupt handler to the
       disk device that gets invoked whenever a new frame is encountered
       while CD audio is being played.

       You must pass in a properly initialized Exec [Interrupt](autodocs-3.5/include-exec-interrupts-h.md) structure
       and be prepared to deal with frame interrupts immediately.
       The interrupt is generated by the exec [Cause](autodocs-3.5/exec-library-cause-2.md) function, so you must
       preserve A6.

       To set up the handler, an [Interrupt](autodocs-3.5/include-exec-interrupts-h.md) structure must be initialized.
       This structure is supplied in io_Data of the CD_ADDFRAMEINT
       command.  The handler then gets linked into the handler chain and
       gets invoked whenever a frame event occurs.  You must eventually
       remove the handler before you exit.

       This command only returns when the handler is removed. That is,
       the device holds onto the IO request until the [CD_REMFRAMEINT](autodocs-3.5/cd-device-cd-remframeint.md) command
       is executed with that same IO request.  Hence, you must use [SendIO()](autodocs-3.5/exec-library-sendio-2.md)
       with this command.
```
   NOTES

       The interrupt handler can be added before or after a play command is
       sent.  Interrupts will only be generated while CD audio is playing.
       Interrupts will not be generated when audio is paused.
   SEE ALSO

```c
       [CD_REMFRAMEINT](autodocs-3.5/cd-device-cd-remframeint.md), [<devices/cd.h>](autodocs-3.5/include-devices-cd-h.md), [<exec/interrupts.h>](autodocs-3.5/include-exec-interrupts-h.md),
       [exec.library/Cause()](autodocs-3.5/exec-library-cause-2.md)
```
