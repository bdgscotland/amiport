Three loops that self-correct.
The adapter layer makes transformations deterministic — one POSIX call maps to one shim call, auditable and traceable. The pipeline runs those transforms through 7 stages with mandatory gates. When something fails, it loops back automatically.

7-STAGE PIPELINE WITH 3 FEEDBACK LOOPS
LOOP 1: BUILD FAILS → TRANSFORM
Missing header, undefined symbol, type mismatch — auto-routes back. If a shim function is missing, /extend-shim writes one. Up to 5 iterations. grep needed 3.
LOOP 2: CRASH → DEBUG AGENT
Guru Meditation on FS-UAE → debug-agent parses Enforcer hit, maps PC to source via addr2line, queries the crash KB, applies fix with git stash rollback, rebuilds. No human. 5 iteration cap.
LOOP 3: REVIEW → REBUILD
Memory-checker finds leak, perf-optimizer spots a hot loop — code goes back through build+test. grep: fgetc()→fgets() delivered 3-5x speedup on 68000.
Enterprise: automated regression, rollback, and fix loops. Not a waterfall that fails silently in UAT.
Why it doesn't fall apart at scale.
A pipeline that works for one migration isn't interesting. A pipeline that works for the 14th — and gets better each time — is what you build a practice on. Three things hold it together: constrained agents, layered enforcement, and a knowledge base that grows.

Three chances to catch every violation. The design choice: exit 2 blocks, exit 0 warns — because blocking agent tool calls would break the pipeline itself.

When the system gets it wrong,
it marks its own mistakes.
Every migration system makes mistakes. The question is whether it buries them or learns from them. Three real stories from this project — each one is now a pattern that prevents the same failure from ever happening again.
THE DEBUNKED PATTERN — PATTERN #9
The pipeline identified exit() as "hanging" on AmigaOS. It built a workaround — a custom _exit() pattern that bypassed atexit handlers — and applied it across ports. The pattern was treated as true.
The actual cause: a single non-ASCII byte in a test script. ARexx — a 1987 scripting language — crashes on any UTF-8 character, including inside comments. A smart-quote in a comment string. That was it.
/* crash-patterns.md */
## Pattern #9: exit() Hang
Status: DEBUNKED

Root cause: ARexx UTF-8 crash, not exit().
_exit() pattern retired.
exit() + atexit() cleanup restored.
Post-mortem: 2026-03-15
This is the proof the system works. It followed a wrong theory, found the real cause, marked its own pattern as debunked, wrote a post-mortem, and corrected all affected ports. The system self-corrects.
THE GREP RE-PORT
During the re-port cycle, FS-UAE testing caught two critical bugs: exit codes always returning 0 (missing braces in if/else) and grep -R silently skipping subdirectories (d_type == DT_DIR — AmigaOS never populates d_type). Both caught before publication.
Both patterns added to known-pitfalls. Source-analyzer now flags braceless multi-statement if/else AND d_type checks. Caught at analysis for all future ports.
THE DESKTOP KILLER
A ported program called fclose(stdin). On Linux, harmless. On AmigaOS, it closes the shell's console handler and crashes the entire Workbench desktop. Not the program — the OS. User reboots.
Now flagged at analysis. Every fclose() that could receive stdin gets caught before any code is written.
