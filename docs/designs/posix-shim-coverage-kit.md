# CEO Plan: POSIX Shim Coverage Kit

Generated by /plan-ceo-review on 2026-03-21
Branch: main | Mode: SCOPE EXPANSION
Repo: bdgscotland/amiport

## Vision

### 10x Check
Transform the amiport POSIX shim from reactive internal infrastructure into a proactive,
well-documented "POSIX Compatibility Kit" — a standalone, community-reusable library
that any Amiga developer can use to port POSIX code without ixemul. Data-driven coverage
analysis ensures the top functions are pre-implemented before ports need them.

### Platonic Ideal
A developer `#include <amiport/posix.h>`, links `-lamiport`, and 80% of POSIX code
Just Works. The other 20% (fork, threads, mmap shared) produces clear compile-time
errors with human-readable messages pointing to redesign patterns. Coverage is tracked
with a dashboard. New functions are added proactively based on corpus analysis, not
discovered mid-port.

## Success Criteria
- READMEs exist for both shim libraries with complete function inventories
- Corpus analysis produces a frequency table for ~30 OpenBSD tools
- Coverage dashboard shows POSIX coverage by category (I/O, string, process, etc.)
- All functions used by 3+ target tools are either: shimmed, available in libnix, or documented as Tier 3
- libnix audit confirms newlib-availability.md is accurate (or updates it)
- `amiport/posix.h` compiles cleanly and includes all Tier 1 + Tier 2 headers
- `AMIPORT_STRICT` mode catches Tier 3 function usage at compile time

## Accepted Scope (added to this plan)
- READMEs for lib/posix-shim/ and lib/posix-emu/
- Convenience header `amiport/posix.h` (includes all Tier 1 headers + Tier 2 via amiport-emu)
- COVERAGE.md generator — extend existing `scripts/shim-coverage.sh` to add POSIX-category-level
  coverage percentages (I/O, string, process, time, directory, signal) parsed from `posix-tiers.md`
- Compile-time Tier 3 guards: `#error` directives in amiport replacement headers (e.g., `amiport/unistd.h`)
  when `AMIPORT_STRICT` is defined. Guards for: fork, exec*, pthread_*, kill, sigaction, mmap(MAP_SHARED)
- `make install-shim` copies headers + .a into `toolchain/sysroot/` (local dir, volume-mounted into Docker)
- libnix audit script (`scripts/audit-libnix.sh`) — clones bebbo/libnix, runs `nm` on .a / scans source,
  compares against `newlib-availability.md`, outputs drift report
- Corpus analysis script (`scripts/analyze-posix-usage.py`) — regex-based (`\b<funcname>\s*\(`) scan of
  ~30 OpenBSD tool source files, cross-references against posix-tiers.md and newlib-availability.md
- Fixture test for corpus analysis (3 small .c files with known calls → expected frequency table)
- ixemul migration guide (`docs/ixemul-migration.md`) — comparison table (ixemul function → amiport
  equivalent) + prose for audience: developers with ixemul-dependent source wanting to switch to -noixemul
- Batch `/extend-shim` execution (SEPARATE SESSION after analysis review)

## Key Implementation Decisions
- Corpus analysis covers ~30 OpenBSD CLI tools (wide scan), regex-based parsing
- Target tools: tr, uniq, tee, basename, dirname, tail, sort, xargs, paste, comm, fold,
  expand, rev, nl, tsort, join, od, strings, fmt, colrm, column, env, printenv, yes,
  true, false, sleep, echo, test, wc, head (+ already-ported: cal, cut, diff, grep, sed)
- Frequency threshold for batch extend: functions used by 3+ target tools AND status=Missing
- Batch /extend-shim runs in a separate session after reviewing analysis results
- OpenBSD source from github.com/openbsd/src (canonical mirror), sparse checkout of usr.bin/
- make install-shim uses local `toolchain/sysroot/` dir mounted into Docker (not image modification)
- Existing port Makefiles keep explicit paths; new ports use sysroot. Migration in future cleanup PR.
- `amiport/posix.h` includes: all headers in `include/amiport/*.h` (Tier 1), plus
  `amiport-emu/*.h` (Tier 2). Does NOT replace existing `amiport/amiport.h` — posix.h is
  the "everything" header, amiport.h is the "core types + macros" header.
- Coverage generator extends existing `scripts/shim-coverage.sh` (adds category breakdown),
  does NOT create a new script.
