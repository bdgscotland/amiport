## Vision

### 10x Check
The 10x version: any Claude instance (or human) can clone this repo and port a Category 1-4 program end-to-end with zero human intervention. The toolkit self-validates (CI catches regressions, doc sync catches drift), self-teaches (port lessons flow back into canonical rules), and self-optimizes (mandatory memory safety, optional perf tuning). Pre-built Docker images mean zero setup friction. The pipeline is a platform, not a script collection.

## Engineering Decisions (from eng review)

### dup()/dup2() fd_table design

```
fd_table:     [BPTR_stdin] [BPTR_stdout] [BPTR_A] [0] ... [BPTR_A]
fd_used:      [1]          [1]           [1]      [0] ... [1]
fd_refcount:  [1]          [1]           [2]      [0] ... [2]  ← shared
fd_closeable: [0]          [0]           [1]      [0] ... [1]
                                          fd=3              fd=N (dup'd)
```

- Parallel `fd_refcount[]` array (not struct change) minimizes churn
- `fd_closeable[]` prevents Close() on process stdin/stdout
- `dup2(file, 1)` makes fd 1 closeable (no longer original stdout)
- Close() only calls AmigaOS Close(fh) when refcount hits 0 AND closeable

### CI bootstrapping
- First run builds Docker image from scratch (~30 min), pushes to GHCR
- Subsequent runs pull cached image (~2 min)
- Self-bootstrapping — no manual first push required

### Hook architecture
- Original/ protection and gcc guard as **native PreToolUse hooks** in settings.json (not hookify)
- Consistent enforcement mechanism for structural guards
- Existing hookify rules (root file creation) remain as-is
