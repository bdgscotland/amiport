## Problem Statement

Port metadata drifts silently — version mismatches between Makefile, PORT.md, .readme, and $VER strings go undetected. TEST-REPORT.md files are generated with empty tables. PORTS.md entries are missing or stale. The user (Duncan) has to manually prompt Claude to fix documentation every time. There is no automated enforcement like there is for agent references (`check-docs`) or test coverage (`check-test-coverage`).

## Solution

A bash validation script (`scripts/check-port-metadata.sh`) that checks every `ports/<name>/` directory for metadata consistency. Wired into `make check-port-metadata`, the pre-commit hook, and CI.

## Validation Checks

For each `ports/<name>/` (skipping `ports/templates/`):

### 1. Required Files
- `Makefile`, `PORT.md`, `<name>.readme`, `test-fsemu-cases.txt` must exist
- At least one `.c` file in `original/`
- At least one `.c` file in `ported/`

### 2. No Template Placeholders
- Grep `PORT.md`, `Makefile`, and `<name>.readme` for `__[A-Z_]+__` patterns
- Any match is a FAIL

### 3. Version Consistency (4 sources)
- `VERSION` variable in `Makefile` (pattern: `VERSION = X.Y`)
- `Version:` line in `<name>.readme` (pattern: `Version:      X.Y`)
- `Version` row in `PORT.md` overview table (pattern: `| Version | X.Y |`)
- `$VER:` string in `ported/*.c` (pattern: `$VER: name X.Y`)
- All four must match
- Exactly one `$VER` string across all `ported/*.c` files (0 or 2+ is FAIL)

### 4. PORTS.md Entry
- Check `PORTS.md` exists at project root (early guard)
- `PORTS.md` must contain a table row with `| <name> |`

### 5. TEST-REPORT.md Quality (if file exists)
- Must not contain "0/0 passed"
- Breakdown table must have at least one data row (not just header)

### 6. Stray Artifacts
- FAIL if any of: `.lha`, `gmon.out`, `*_native`, `*.map` files exist in port directory

## Output Format

```
=== Checking port metadata consistency ===
grep: required files ........... PASS
grep: no placeholders .......... PASS
grep: version consistency ...... FAIL (Makefile=1.68, PORT.md=1.0)
grep: PORTS.md entry ........... PASS
grep: test report quality ...... WARN (0/0 passed)
grep: no stray artifacts ....... FAIL (gmon.out, grep-1.68.lha)

FAILED: 1 port(s) with errors, 1 with warnings
```

Non-zero exit on any FAIL.

## Test Strategy

### Real port validation
Run against all 7 existing ports. Known bugs:
- grep: PORT.md version mismatch (1.0 vs 1.68), gmon.out, .lha, .map stray artifacts

### Synthetic test fixture
Create `tests/fixtures/bad-port/` with deliberate errors:
- Missing .readme
- Template placeholder in PORT.md
- Version mismatch across sources
- No $VER string
- Multiple $VER strings
- Stray gmon.out

- `make check-port-metadata` — new Makefile target
- `.githooks/pre-commit` — add after `check-docs`
- `.github/workflows/ci.yml` — add step after existing checks
- `CLAUDE.md` — document new target in Build Instructions
- `README.md` — add to make targets list

## Decisions Made

1. Stray artifacts **block** commits (not just warn)
2. Exactly one `$VER` string required per port (catches duplicates)
3. Synthetic test fixture for regression testing
4. test-fsemu.sh report fix included in scope
5. Examples directory excluded (no .readme or PORTS.md entries)
