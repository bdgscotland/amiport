# test-fsemu-cases.txt -- FS-UAE functional test suite for col 1.20
# col: filter reverse line feeds from input (OpenBSD col.c rev 1.20)
# Category: 1 (CLI tool)
# Minimum tests required: 8
#
# Flags tested: -b, -f, -h, -l num, -x
# Exit codes:
#   exit(0)  -- normal completion (RETURN_OK)
#   exit(10) -- bad -l argument (errx, RETURN_ERROR)
#   exit(10) -- unknown flag (usage(), RETURN_ERROR)
#   exit(10) -- extra file argument given (usage(), RETURN_ERROR)
#   exit(10) -- stdout write error (err, RETURN_ERROR)
#
# NOTE: col reads from STDIN_FILENO only -- no file arguments accepted.
# All tests that need stdin input use WORK:test-col-run.rexx, an ARexx
# wrapper that creates an Execute script with < redirection internally.
# This avoids prohibited < and | in CMD lines (ARexx ADDRESS COMMAND
# limitation) while still feeding data to col.
#
# NOTE: Error messages go to stderr -- not captured by test harness.
# Error path tests use EXPECT_RC: only.
#
# NOTE: Some tests produce output containing backspace bytes (0x08).
# These tests use EXPECT_CONTAINS: with a clean substring rather than
# EXPECT: with raw control chars that are hard to embed in test files.
# Tests using -b produce clean ASCII output and use EXPECT: (exact match).
#
# Native command used to compute all EXPECT values:
#   col < inputfile | head -1
# (macOS col is BSD-derived; OpenBSD behavior verified against source)
