# test-fsemu-visual-cases.txt -- Visual verification tests for mg 3.7
# Runs in a SEPARATE FS-UAE pass from functional tests (ADR-024).
# Requires forked FS-UAE with per-unit ANSI capture.
#
# SCOPE: CMD_WRITE captures initial file display (the file load path
# that writes all lines to the console on open). Interactive echo
# (typed chars, cursor movement) is NOT captured by CMD_WRITE --
# mg renders those through a different console.device path.
#
# ADR-025 adds SCREEN_READ + EXPECT_TRAP_CURSOR for cursor position
# verification via a custom FS-UAE trap that reads ConUnit fields directly.
# EXPECT_TRAP_CURSOR reads from .screen JSON (ConUnit trap, authoritative
# for interactive ops). EXPECT_CURSOR reads from pyte (ANSI reconstruction,
# static display only).
#
# Tests 1-4: file content display (EXPECT_AT, ADR-024)
# Tests 5-6: cursor movement (EXPECT_TRAP_CURSOR, ADR-025)

# ============================================================
# 1. Basic file -- all 5 lines verified
# ============================================================

ITEST: Visual: basic file content correct on all 5 lines
LAUNCH: WORK:mg -n WORK:test-mg-basic.txt
KEYS: WAIT3000
SCRAPE
EXPECT_AT 1,1,Hello, Amiga world!
EXPECT_AT 2,1,This is a basic test file for mg.
EXPECT_AT 3,1,Line three has some text.
EXPECT_AT 4,1,Line four: the quick brown fox jumps over the lazy dog.
EXPECT_AT 5,1,Line five: testing cursor movement and search.

# ============================================================
# 2. Scroll file -- verify different file produces different output
# ============================================================

ITEST: Visual: scroll file displays numbered lines
LAUNCH: WORK:mg -n WORK:test-mg-scroll.txt
KEYS: WAIT3000
SCRAPE
EXPECT_AT 1,1,Line 001: The Amiga 1200 was released in 1992.
EXPECT_AT 3,1,Line 003: The AGA chipset brought 256 colours on screen.
EXPECT_AT 5,1,Line 005: AmigaOS uses a microkernel called Exec.

# ============================================================
# 3. Read-only mode -- -R flag doesn't prevent display
# ============================================================

ITEST: Visual: -R read-only mode still displays file
LAUNCH: WORK:mg -n -R WORK:test-mg-basic.txt
KEYS: WAIT3000
SCRAPE
EXPECT_AT 1,1,Hello, Amiga world!
EXPECT_AT 4,1,Line four: the quick brown fox jumps over the lazy dog.

# ============================================================
# 4. Read-only mode -- verify -R flag still displays content
# (different flag combination than test 1)
# ============================================================

ITEST: Visual: -R flag displays all 5 lines
LAUNCH: WORK:mg -n -R WORK:test-mg-basic.txt
KEYS: WAIT3000
SCRAPE
EXPECT_AT 1,1,Hello, Amiga world!
EXPECT_AT 5,1,Line five: testing cursor movement and search.

# ============================================================
# 5. Cursor movement with C-n (trap proof -- decoupled from CSI fix)
# ADR-025: Proves trap pipeline works using known-good CTRL_N
# ============================================================

ITEST: Visual: cursor moves down with C-n (trap proof)
LAUNCH: WORK:mg -n WORK:test-mg-scroll.txt
KEYS: WAIT2000,CTRL_N,WAIT1000
SCRAPE
SCREEN_READ
EXPECT_TRAP_CURSOR 2,1
CLEANUP: CTRL_X,WAIT300,CTRL_C
EXPECT_RC: 0

# ============================================================
# 6. Cursor movement with DOWN arrow (CSI fix proof)
# ADR-025: Proves CSI 0x9B to ESC [ translation + trap together
# ============================================================

ITEST: Visual: cursor moves down with arrow key (CSI fix proof)
LAUNCH: WORK:mg -n WORK:test-mg-scroll.txt
KEYS: WAIT2000,DOWN,WAIT1000
SCRAPE
SCREEN_READ
EXPECT_TRAP_CURSOR 2,1
CLEANUP: CTRL_X,WAIT300,CTRL_C
EXPECT_RC: 0
