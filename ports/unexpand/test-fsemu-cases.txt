# test-fsemu-cases.txt -- FS-UAE functional test suite for unexpand 1.13
# unexpand: convert leading spaces to tabs (inverse of expand)
# Source: OpenBSD unexpand.c,v 1.13 2016/10/11 16:22:15 millert
# Category: 1 (CLI tool)
# Minimum tests required: 15
#
# Flags tested: -a (convert all spaces, not just leading)
# NOTE: This port does NOT support -t (not in OpenBSD 1.13 source).
#       Any flag other than -a returns RC=10.
#
# Error paths tested:
#   fprintf(stderr, "usage: unexpand [-a] [file ...]\n") + exit(10) -- unknown flag
#   perror(argv[0]) + exit(10) -- nonexistent file
#
# NOTE: stderr is NOT captured by the harness.
#       Error path tests use EXPECT_RC: only.
#
# STDIN HANG WARNING: unexpand reads stdin when no file arg is given.
#   All CMD lines pass an explicit file argument to prevent hang.
#   Tests for stdin behavior are omitted (ARexx limitation -- no EOF injection).
#
# Tab characters in EXPECT: lines are literal ASCII 0x09.

# ============================================================
# PRECISION TEST: verify tab stop arithmetic is correct
# ============================================================

# Native: unexpand -a test-unexpand-allflag.txt | head -1
# "hello        world" -- 8 spaces at col 5 reaches col 13
# After "hello" (ocol=5), 8 spaces -> dcol=13
# Tab emission: ((5+8)&~07)=8 <= 13? YES. Not (5+1==13). Emit tab. ocol=8.
# ((8+8)&~07)=16 <= 13? NO. Stop.
# Fill: 8 < 13 -> 5 spaces. Result: "hello\t     world"
TEST: Precision -- tab stop arithmetic at non-aligned column position
CMD: WORK:unexpand -a WORK:test-unexpand-allflag.txt
EXPECT: hello	     world
EXPECT_RC: 0
